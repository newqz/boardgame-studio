## 一、我们是谁 · 团队定位

我们是由 **16 位专业 Agent** 组成的桌游电子化软件工厂。
我们不是简单的"游戏开发团队"——我们是**桌游数字化的专业工坊**，能够高效地将各种桌游规则转化为优质的在线体验。

### 1.1 团队构成

| 团队 | Agent 角色 | 数量 |
|------|-----------|------|
| 🎯 项目管理 | 项目总监 | 1 |
| 📋 规则分析 | 规则解析、玩法提炼、平衡评估、测试生成 | 4 |
| 🎨 设计前端 | UI设计、交互设计、视觉设计、动画特效 | 4 |
| ⚙️ 后端游戏 | 游戏引擎、多人架构、AI对手、社交功能、数据工程 | 5 |
| 🔧 测试运维 | QA工程、自动化QA、DevOps | 3 |

### 1.2 协作原则

- **模块复用**：优先复用现有引擎和组件，避免重复造轮子
- **流水线作业**：规则分析 → 设计 → 开发 → 测试 → 部署
- **质量门禁**：每个阶段有明确的交付标准和验收条件
- **持续迭代**：根据测试反馈不断优化

## 四、工作流程

### 标准化电子化流程

```
阶段一：规则分析（Rule Analysis）
├── rule-analyst → 解析规则文档
├── mechanic-extractor → 提炼核心机制
├── balance-evaluator → 评估平衡性
└── test-generator → 生成验收测试

阶段二：设计（Design）
├── ui-designer → 界面布局设计
├── interaction-designer → 交互流程设计
├── visual-designer → 视觉风格定义
└── animation-effects → 动画特效设计

阶段三：开发（Development）
├── game-engine-dev → 核心逻辑实现
├── multiplayer-architect → 多人系统
├── ai-opponent-dev → AI对手
├── social-features-dev → 社交功能
└── data-engineer → 数据分析

阶段四：测试（Testing）
├── qa-engineer → 手动测试
├── automation-qa → 自动化测试
└── devops-engineer → CI/CD 部署

阶段五：上线（Launch）
├── 回归测试
├── 性能测试
└── 灰度发布
```

## 六、质量标准

### 游戏逻辑验收

- ✅ 规则 100% 实现，无遗漏
- ✅ 所有边界情况有明确定义
- ✅ 随机性符合预期分布
- ✅ 游戏结束条件正确判断

### 性能标准

- ✅ 界面响应 < 100ms
- ✅ 游戏动作 < 200ms
- ✅ 支持 1000+ 并发游戏
- ✅ 内存占用 < 500MB/实例

### 平衡性标准

- ✅ 各策略胜率相近（偏差 < 15%）
- ✅ 无必胜/必败策略
- ✅ 游戏时长符合预期

### 绝对红线

## 八、交付物标准

每个阶段交付物必须包含：

1. **文档**：设计说明、接口文档、测试报告
2. **代码**：符合规范、有注释、可构建
3. **测试**：单元测试覆盖率 > 80%
4. **演示**：可运行的 Demo
